Fiero 1227170 Tuning

This writeup is for those who wish to tune their original 1227170 ECU out of their Fiero, with the L44 engine from 1986-1988. Going into this project, I had not really found much information out there on how to make a modern process to tune my 1988 Fiero Formula easily, so instead of gatekeeping the information, I decided to provide this writeup, because if I had this when I first rebuilt my Fiero, I would have driven a lot more often. This writeup assumes you have some knowledge of how to use a computer, wire up components on a breadboard, and the ability to solder.

As usual, any aftermarket modifications you make could have unintended consequences, of which you will be responsible for.

Materials needed:

1x AT28C256-15su

1x Arduino Mega or clone

2x 4.7K Ohm resistors

1x 28 pin DIP socket

1x Row of 2.54mm header pins (most common 40 pin variant is acceptable)

1x Toggle switch

Computer with Windows installed

Assorted wires

Either prototype PCBs or order the PCB from the gerber files provided

The Chip

To start out, you will need an EEPROM chip to program. The most viable and easy (modern) alternative out there is for the 2732 that is found in the Fiero ECU is that of the Atmel at28c256-15su, a 32K EEPROM that does not require a special programmer or a UV eraser, and can store technically 8 different tunes! This chip is almost pin compatible with the original chip, and the extra few pins are just used for the extra 28K of data the chip can store, and a control line since it is an EEPROM instead of an EPROM. This also means that tunes can almost indefinitely be reprogrammed within a few minutes. The provided documentation is from my prototype board, which only switches between two, a stock tune, and a performance tune. AT28C256’s are readily available at many different online stores, such as Mouser, Digikey, and Amazon.

The Tune

In order to make the tune files you want, you will need to download DOSbox, a DOS emulator. This is due to the fact that most tune programs are behind paywalls and proprietary software that does not make itself novice-friendly. DOSbox can be found at https://www.dosbox.com/download.php?main=1, where there install should be like any other program, nothing special. The tuning program that will be used in GMEPro (by Terry Kelly), with the definition file, base tune, and a street tune provided at the link: https://github.com/AnthonyClemens/1227170-Tuning/blob/main/GMEPRO.zip

Once extracted and DOSbox installed, launch DOSBox and type in the commands “MOUNT C “ and then the directory in which your GMEPro is extracted to (Ex. C:\Users\anthony\Downloads\GMEPRO, see screenshot below). Type “C:” and then “gmepro.exe”, and GMEPro will launch.

Press the Esc key on your keyboard and you will be prompted with the main menu for tuning. Since this program is running in DOS, you will be using the keyboard for all of your tuning.

First, load the EPE provided, labeled “868FIERE.EPE”, and then load the BIN labeled “88FIERO.BIN” or the other BIN provided. Once those are loaded, you can edit the tables, thus creating your tune. If you would like to enable EGR once the car is “FULLY” warmed up, the EGR Enable Temperature would be set to 304 degrees Fahrenheit, so once your engine gets to a warm and comfortable 304 degrees, your EGR will kick in ;). Once you have made the changes you wish to make, you will be able to save the binary to your computer. 

Combining Tune Files

If you do not wish to have two tune files, you may skip this step and there will be no repercussions to doing only one tune.

To combine the tune files, I used HxD, found at https://mh-nexus.de/en/downloads.php?product=HxD20, once installed, open the program and go to tools, then file tools, then Concatenate.

Select your two bin files you wish to switch between and then name the output file you want to combine them into.

Making the Programmer

If you have an EEPROM programmer that already can write the AT28C256, then you can skip this step and use your programmer. If you are like me and just had an Arduino Mega laying around, then this will help you. If you don’t already have an Arduino Mega, you can buy one at the link: https://store-usa.arduino.cc/collections/boards-modules/products/arduino-mega-2560-rev3, or if you don’t care about brand, then you can find an Elegoo clone on Amazon for significantly cheaper and already comes with wires and breadboard for you to wire up the programmer: https://a.co/d/4MNtx9X. You will also need 2x 4.7k ohm resistors, one for the programmer and one for the adapter board, these can also be found at almost any online retailer. You may either wire the programmer out on a breadboard, or buy one from Ferret Labs, the original designers of this programmer, at: https://www.tindie.com/products/oddblk/simple-eeprom-programmer-shield-for-arduino-mega/.Once you have your Arduino Mega and breadboard out, wire up the board like shown here, modified from “oddblk” on Github, of whose code and diagrams I modified for this project:

First, you must program the Arduino Mega with the .ino file to correctly interface with your computer. The software for the Arduino ide can be found at https://www.arduino.cc/en/software, and once you have downloaded and installed it, you can download the program that the Arduino will be running at https://github.com/AnthonyClemens/eeprom-writer-AT28C256/, which also has the wiring diagram and subsequent explanation on the code modification done on the original code. Once you download that .ino file and double click it, Arduino IDE will ask if you want it to put itself in a folder, click “ok”. Once the file is open, ensure that your board shows up like shown below:

To program the software onto the Arduino, click the right arrow near the top left of the screen. If it is successful, you will see only white text in the “Output” at the bottom of the screen, if it fails, there will be red text indicating what went wrong, and troubleshooting will have to be conducted from there.

Programming the Chip

In order to program the chip, you will need this program: http://danceswithferrets.org/geekblog/wp-content/uploads/2017/11/eewriter.zip, along with the .bin file created earlier. Once you have extracted and launched EEwriter, it should automatically detect the Arduino, you can confirm by clicking “Test”, and if it says “Tested OK!”, then you have done everything correctly so far. Atmel chips have by default a Software Data Protection (SDP), so that once changes are made, you can lock the chip from being written to.

Speaking of SDP, you will have to click on tools, and then “Unprotect (Atmel)”. You may now click “File” and then “Load ROM Image …”, and select the .bin file from earlier. EEwriter should automatically detect that the size of the file is 8KB (64kb), and then you should be able to click “Write EEPROM”. Once it completes successfully, you MUST click “Verify EEPROM”, in order to make sure that the SDP was disabled and the write went through correctly. Now, click back on tools, and then select “Protect (Atmel)”, this will make sure that the data on your chip will not be overridden by accident. You may now remove the chip from the breadboard, and your AT28C256 is now programmed with both tunes on it.

Making the Adapter

The adapter we are making has a wiring diagram as shown below. R1 can be replaced with the 4.7K Ohm resistors that you bought for the programmer. The basic explanation of the diagram below is that all of the extra address pins are pulled to ground, except A12, which has a pulldown resistor, meaning that if connector P1 is shorted, such as via a toggle switch, it will pull A12 high, accessing the second tune that you programmed in HxD earlier, whereas if you programmed one tune to the chip, then shorting P1 will cause your car to not function as intended. As far as I know it doesn’t hurt the car if you accidentally do and start the car, it will just sit at 1,500rpm or so and not do anything else. Make sure the check engine light comes on when you put the key in run, this means that ECU detects the tune chip correctly and is ok to start.

If you have no desire to make your own adapter board, linked here is gerber files to a board I designed based on the above schematic. All you need to do is solder the 28 pin socket, pulldown resistor, and the 2.54mm pitch header pins: https://github.com/AnthonyClemens/1227170-Tuning/blob/main/Gerber-EEPROM-adapter.zip.

If you want to make your own, I used a basic double sided prototype PCB and staggered the DIP socket and the header pins, using a healthy amount of solder to bridge the connections from the neighboring pads.

Once your adapter is done, you may wire up your switch to P1 if you opted to have two tunes, and then it should be as simple as removing your old PROM and installing the new adapter board. Be aware, that more than likely the new adapter will be too tall to reinstall the ECU cover, and the chip will want to fall out on its own due to using pin headers, so what I did was put the plate back on, and use the original bolts and cover to apply light pressure to the adapter since it has a foam cover on the inside of the plate. Run your wires with your toggle switch to wherever you want, probably the easiest would be near the ALDL port. Then you should be done! Make sure that when you are switching tunes you have the car fully off and the engine is not running.
